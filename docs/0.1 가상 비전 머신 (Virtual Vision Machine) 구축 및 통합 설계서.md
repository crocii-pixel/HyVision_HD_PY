# 가상 비전 머신 (Virtual Vision Machine) 구축 및 통합 설계서

## 1. 설계 개요 및 철학 (Architecture Philosophy)

본 가상 비전 머신(이하 VVM)은 실제 비전 하드웨어(OpenMV 등)와 **100% 동일한 프로토콜(32B/64B)과 동일한 논리 엔진**을 PC 메모리 상에 소프트웨어적으로 구현한 '디지털 트윈(Digital Twin)'입니다.

- **완벽한 WYSIWYG 제공:** TEACH 모드에서 사용자가 파라미터를 변경할 때마다 VVM이 실시간 연산하여 화면에 결과를 표출합니다. 이 결과는 실제 하드웨어에 이식되었을 때의 결과와 수학적으로 완전히 일치합니다.
- **하드웨어 제약 탈피:** 카메라의 연산 능력이 부족할 경우, 장비는 이미지 획득(Streaming) 역할만 수행하고 무거운 비전 연산(Pattern Matching, FND 등)은 PC의 VVM이 전담하는 '호스트 연산 모드'로 매끄럽게 전환할 수 있습니다.

## 2. 핵심 기능 1: 이중 이미지 소싱 (Dual Image Sourcing)

VVM은 연산을 수행할 원본 프레임(이미지)을 얻어오는 출처를 두 가지 모드로 완벽히 분리 지원합니다. `ImageProvider` 인터페이스를 통해 VVM의 코어 로직은 이미지가 어디서 왔는지 알 필요 없이 연산에만 집중합니다.

### 모드 A: 실시간 하드웨어 소싱 (Physical Camera Mode)

- **개념:** 실제 OpenMV나 일반 USB 웹캠 등 물리적 장치로부터 실시간 프레임을 획득합니다.
- **동작 시나리오:** * 하드웨어가 연산 없이 `LIVE` 스트리밍 모드로 이미지만 PC로 전송합니다.
  - VVM이 이 실시간 이미지를 낚아채어(Intercept) 내부에 세팅된 비전 툴(HyLine, HyLocator 등)을 통과시킵니다.
  - 실제 라인에 장비를 설치한 상태에서 PC의 압도적인 연산 능력을 활용하고 싶을 때 사용합니다.

### 모드 B: 로컬 스토리지 소싱 (Virtual Folder Mode)

- **개념:** PC에 저장된 특정 폴더의 이미지들이나 단일 파일들을 반복적으로 읽어 들입니다.
- **동작 시나리오:**
  - 장비가 물리적으로 연결되어 있지 않은 오프라인(사무실/노트북) 환경에서 작동합니다.
  - 현장에서 발생한 '불량(NG) 이미지 샘플' 수백 장을 폴더에 넣고 VVM을 돌려, 새로 수정한 레시피(설정값)가 불량을 정확히 잡아내는지 일괄 검증(Batch Test)할 수 있습니다.

## 3. 핵심 기능 2: 코어 비전 엔진의 OpenCV 포팅 (Logic Porting)

펌웨어(MicroPython)의 제한된 비전 라이브러리 연산을 PC(CPython)의 막강한 `OpenCV` 및 `NumPy` 기반으로 이식하여 수학적 동등성(Math Parity)을 확보합니다.

- **`HyLine` (선 추출):** * MicroPython `img.get_statistics().mean()` ➔ NumPy 슬라이싱 `np.mean(img[y:y+h, x:x+w])`로 대체.
  - 픽셀 루프를 C 기반의 벡터 연산으로 고속화.
- **`HyPatMat` (템플릿 매칭):** * `img.find_template` ➔ OpenCV `cv2.matchTemplate(..., cv2.TM_CCOEFF_NORMED)` 및 `cv2.minMaxLoc`으로 완벽히 1:1 대응.
- **역방향 크롭 (Inverse Crop Warping):**
  - `sensor.alloc_extra_fb()` ➔ OpenCV `cv2.getRotationMatrix2D` 및 `cv2.warpAffine`을 활용한 서브픽셀 단위의 정밀 회전/크롭 구현.

## 4. 핵심 기능 3: 가상 프로토콜 파이프라인 (Virtual Protocol Bus)

UI(`MainWindow`) 코드를 수정하지 않기 위해, 실제 시리얼 포트와 동일하게 행동하는 **메모리 기반 가상 큐(Virtual Queue Bus)**를 구축합니다.

1. **명령 수신부:** UI가 `SET_TOOL (0x10)` 등의 32바이트 Command를 VVM 입력 큐에 넣습니다.
2. **상태 머신 (State Machine):** VVM 내부의 가상 펌웨어 루프가 이를 해석하여 `LIVE`, `TEACH`, `TEST` 모드를 전환합니다.
3. **결과 송신부:** 연산이 끝나면 VVM은 결과를 **장치와 완벽히 똑같은 형태의 64바이트 1차원 배열(Burst)**로 패킹(Packing)하여 출력 큐로 던집니다.
4. **투명성:** UI의 `RecipeManager`는 이 패킷이 USB 선을 타고 온 것인지, PC 메모리에서 온 것인지 구분하지 못하며 동일하게 렌더링합니다.

## 5. 단계별 구현 및 통합 로드맵 (WBS)

- **Phase 1: VVM 코어 프레임워크 작성**
  - `VirtualVisionEngine.py` 스레드 생성.
  - Image Sourcing (모드 A/B) 선택기 및 프레임 버퍼 로직 구현.
- **Phase 2: 비전 툴 OpenCV 이식 및 테스트**
  - PC 버전 `HyLine`, `HyLocator`, `HyIntersection` 클래스 작성.
  - 크리스(QA)의 수학 오차 검증(Math Parity Test) 진행.
- **Phase 3: 가상 통신망 연동**
  - `OpenMVWorker` 내부에 분기를 두어, 대상 포트가 'VIRTUAL_FOLDER' 또는 'VIRTUAL_CAM'일 경우 Serial I/O 대신 Queue I/O를 수행하도록 어댑터(Adapter) 패턴 적용.
- **Phase 4: UI 티칭 실시간 동기화**
  - TEACH 모드에서 ROI 박스 마우스 드래그 이벤트 발생 시, 즉시 32B 패킷을 VVM으로 전송하여 60FPS 급의 실시간 연산 피드백 UI 구현.
