# UI 설계 및 사용 시나리오 명세서

**버전:** 1.0
**최종 수정:** 2026-04-15
**참조:** 마스터 상세서 v4.3, 프로토콜 명세서 v2.0, 클래스 명세서 v2.0

---

## 1. 설계 철학

### 1.1. 3대 원칙

1. **캔버스 우선 (Canvas-First):** 검사 이미지는 항상 최대 영역. 설정 UI는 영구 잠식 불가.
2. **직접 조작 (Direct Manipulation):** ROI는 드래그(이동·리사이즈·**회전**)로 직접 편집. 트리 조작은 드래그앤드롭 우선.
3. **모드 단절 (Modal Clarity):** LIVE/TEACH/TEST/RUN 각 모드는 UI 구성이 명확히 다름.

### 1.2. DCOM 아키텍처 이해 (UI 설계 전제)

- **PC = 순수 UI/편집 계층.** PC 자체는 DCOM 실행 장치가 아님.
- **VVM(Virtual Machine)** = PC 위에서 돌아가는 DCOM 참여 장치. `exec_dev_id` 보유.
- **공유 Executor Tree:** 모든 장치(물리 카메라 + VVM)가 논리적으로 동일한 트리를 보유.
  각 장치는 `exec_dev_id`가 자신인 노드만 실행, 나머지는 바톤 터치.
- **UI의 TEACH/TEST 구분** = 사용자 편집 관심사 분리. 실행 계층(exec_dev_id)과 무관.

### 1.3. 패널 배치 정책

| 패널 | 위치 | 가시성 |
|------|------|--------|
| `VisionCanvas` | 중앙, 창 크기 확장 | 항상 |
| `TreePanel` | 좌측 Dockable | 접기/펼치기 |
| `OverlayPanel` | 캔버스 위 플로팅 | 드래그·투명도 조절·핀 |
| 로그 패널 | 하단 Dockable | 접기/펼치기 |

## 2. 4대 모드 구성

### 2.1. 모드 전환 흐름

```
LIVE ──▶ TEACH ──▶ TEST ──▶ RUN
 ↑          │설정/검증   │설정/검증
 └──────────┴────────────┘
```

- **LIVE:** 카메라 HW 셋업. 서브모드 없음.
- **TEACH:** 비전툴 등록·편집. 서브모드: 설정 / 검증.
- **TEST:** 로직 조립·판정 설정. 서브모드: 설정 / 검증.
- **RUN:** 양산 실행. 서브모드 없음, 편집 전면 잠금.

### 2.2. 서브모드 공통 동작

| 서브모드 | 이미지 | CRUD | 실행 |
|---------|--------|------|------|
| **설정(Setup)** | 단 1프레임 고정 | 가능 | UI Preview Engine (실시간 피드백) |
| **검증(Verify)** | 동영상 스트림 | 잠금 | VVM / 물리 장치 |

- 검증 서브모드에서 ROI를 조정(이동·리사이즈·회전)한 경우 → 설정 서브모드로 돌아올 때 **그대로 반영**.

## 3. TEACH 모드 상세

### 3.1. 레이아웃

```
┌─────────────────────────────────────────────────────────────┐
│ [LIVE] [TEACH▼] [TEST] [RUN]  ● 설정  ○ 검증               │
├──────────────┬──────────────────────────────────────────────┤
│              │  VisionCanvas                                 │
│ 비전툴 리스트 │                                              │
│ ─────────── │  ┌─────────────────────┐ ← 플로팅 OverlayPanel│
│ #1 📏 HyLine │  │ ☰ HyLine #1  📌🔅✖ │  (드래그, 반투명)    │
│ #2 📏 HyLine │  │ num_splits [5]     │                      │
│ #3 🎯 HyLPM  │  │ cut_ratio  [0.55]  │                      │
│ #4 📊 HyDist │  └─────────────────────┘                      │
│ #5 📊 HyFND  │                                              │
│              │  [◀ 이전 프레임]  [새 프레임 요청 ▶]          │
│ [+ 비전툴 ▼] │                                              │
│ [라이브러리] │                                              │
│ ──────────── │                                              │
│   ◀ 접기    │                                              │
└──────────────┴──────────────────────────────────────────────┘
```

### 3.2. 비전툴 생성 (직접 삽입)

```
[+비전툴 ▼] 클릭
  → 드롭다운: HyLine / HyLocator / HyPatMat / HyIntersection /
              HyLinePatMat / HyDistance / HyContrast / HyFND
  → 선택 → 캔버스 커서 십자선 전환
  → 드래그로 ROI 그리기 (HyFND는 4-Step 위저드로 전환)
  → 완료: 리스트 하단에 새 항목 추가 + OverlayPanel 자동 출현
  → SET_TOOL 32B 전송 → UI Preview Engine 즉시 연산 → 렌더링
```

### 3.3. ROI 편집 핸들 (캔버스)

```
   ┌──○──┐    ○ = 리사이즈 핸들 (모서리·중간)
   │     │    ↺ = 회전 핸들 (상단 중앙에서 오프셋)
   │  ✛  │    ✛ = 이동 (박스 내부 드래그)
   └──○──┘
      ↺
```

- **이동:** ROI 내부 드래그
- **리사이즈:** 8방향 핸들 드래그
- **회전:** 상단 중앙 위 오프셋 핸들 드래그 (각도 1° 단위, Shift 누르면 15° 스냅)
- 모든 조작 시 `SET_TOOL` 실시간 전송 → UI Preview Engine 60FPS 피드백

### 3.4. 툴 내 다른 툴 참조 (콤보 드롭다운)

- `HyLocator` OverlayPanel → `target_tool` 필드: 드롭다운 → 리스트 내 HyLine/HyPatMat 목록
- `HyLinePatMat` OverlayPanel → `target_line` 필드: 드롭다운 → 리스트 내 HyLine 목록
- `HyDistance` OverlayPanel → `source_a`, `source_b`: 드롭다운 → 전체 툴 목록
- `HyIntersection` OverlayPanel → `source_a`, `source_b`: 드롭다운

**규칙:** 드롭다운 선택 시 리스트에서 참조 대상 툴 하이라이트. 참조 대상이 자신보다 아래(나중 실행)이면 ⚠ 경고.

### 3.5. 비전툴 실행 순서

- 리스트 순서 = 실행 순서 (위→아래)
- 드래그앤드롭으로 리스트 항목 순서 변경 가능
- **HyLocator는 항상 리스트 최상단 권장** (다른 툴이 Fixture 의존)
- 의존성 위반 시 ⚠ 표시 (실행은 허용)

### 3.6. LIVE 모드

```
[LIVE▼] 모드 → 트리 패널 자동 접힘
→ 캔버스: 원시 영상 스트리밍 (비전 오버레이 OFF)
→ 카메라 설정 플로팅 패널 출현 (노출·게인·화이트밸런스)
→ [저장] 없이 이탈 → 원래 설정값으로 Rollback
```

## 4. TEST 모드 상세

### 4.1. 레이아웃

```
┌─────────────────────────────────────────────────────────────┐
│ [LIVE] [TEACH] [TEST▼] [RUN]  ● 설정  ○ 검증               │
├────────────────────┬────────────────────────────────────────┤
│                    │  VisionCanvas                           │
│ 비전툴 리스트       │                                        │
│ (드래그 전용, 읽기)  │  ┌──────────────────────┐ ← 플로팅    │
│ #1 📏 HyLine       │  │ ☰ HyAnd #10 [Fin] ✖ │              │
│ #2 📏 HyLine       │  │ ┌ 로직툴 ┐ ┌ HyFin ┐ │              │
│ #3 🎯 HyPatMat     │  │ AND 설정  I/O 매핑  │ │              │
│ ────────────────   │  └──────────────────────┘              │
│ 로직 트리           │                                        │
│ ────────────────   │  OSD: Status Box / Spec Box (드래그)   │
│ ▼ 📦 HyAnd [Fin]   │                                        │
│   🔗 #1 Line OK    │                                        │
│   🔗 #2 Line OK    │                                        │
│   ▼ ⏱ HyWhen      │                                        │
│     🔗 #3 Pat OK   │                                        │
│                    │                                        │
│ [+ 로직툴 ▼]       │                                        │
└────────────────────┴────────────────────────────────────────┘
```

### 4.2. 로직 트리 구성 규칙

| 규칙 | 내용 |
|------|------|
| **루트 노드** | 로직툴만 가능 (HyAnd / HyOr / HyWhen) |
| **비전툴 배치** | 로직툴 내부에만 (참조, 드래그앤드롭) |
| **로직툴 중첩** | 무제한 허용 (HyAnd 내부에 HyOr 등) |
| **비전툴 다중 참조** | 동일 tool_id를 여러 로직 노드에서 참조 가능 (실행은 TEACH에서 1회) |
| **로직툴 복사** | 없음. 항상 신규 생성 (새 tool_id 할당) |
| **빈 로직툴** | 허용. ⚠ 아이콘 표시. 빈 AND=True, 빈 OR=False |

### 4.3. HyFin 처리

- HyFin은 별도 트리 노드가 아님
- 로직 노드(HyAnd/HyOr)에 `[Fin]` 배지 부착 → 해당 로직 결과를 HyFin으로 출력
- 노드 클릭 → OverlayPanel에 **탭 2개**: `[로직툴 설정]` / `[HyFin 출력 설정]`
- 여러 노드에 `[Fin]` 부착 가능 (다중 출력)

### 4.4. 비전툴 리스트 → 로직 트리 드래그앤드롭

```
TEACH 비전툴 리스트 (상단, 읽기 전용)
    │
    │ 드래그
    ▼
로직 트리 내 특정 로직 노드에 드롭
    → 🔗 참조 노드 생성 (tool_id 저장)
    → 동일 툴을 다른 로직 노드에도 드롭 가능
```

**참조 노드(🔗) 속성:**
- 읽기 전용 (파라미터 편집은 TEACH에서)
- 클릭 → OverlayPanel에 "TEACH에서 편집" 버튼만 표시
- 더블클릭 → TEACH 모드로 전환 + 해당 툴 하이라이트

### 4.5. 로직툴 생성

```
[+ 로직툴 ▼] 클릭
  → 드롭다운: HyAnd / HyOr / HyWhen
  → 선택 → 트리 최하단 루트 레벨에 새 노드 추가
  → OverlayPanel 자동 출현 (HyWhen의 경우 watch_tool_id 드롭다운)
  → [Fin] 배지 부착 토글 버튼 OverlayPanel 상단에 제공
```

### 4.6. OSD 설정

- **Status Box, Spec Box:** 캔버스에 기본 위치로 표시 → 드래그로 이동
- 클릭 시 플로팅 패널: 폰트 크기, 색상, 표시 여부 토글
- Decorations(ROI 박스 색상·십자선 등): OverlayPanel 하단 탭

### 4.7. 트리 우클릭 컨텍스트 메뉴

- 📋 복사 (`Ctrl+C`) / 📄 붙여넣기 (`Ctrl+V`) — 로직툴 노드 대상
- 💾 템플릿으로 저장
- ❌ 삭제 (`Del`) — 참조 포함 경고 다이얼로그
- ↔ 이동 → (다른 로직툴 목록 서브메뉴)

## 6. OverlayPanel 상세

### 6.1. 공통 헤더

```
┌──────────────────────────────────────────────────┐
│ ☰  [HyLine #1]              📌  🔅━━●━━  ✖     │
├──────────────────────────────────────────────────┤
│ (툴 타입별 동적 폼)                               │
└──────────────────────────────────────────────────┘
```

- `☰` 드래그 핸들: 위치 이동, 위치는 `QSettings` 저장
- `📌` 핀: 다른 툴 선택 시 유지 (최대 4개 동시)
- `🔅━━●━━` 투명도 슬라이더 (alpha 0.3~1.0)
- `✖` 닫기

### 6.2. HyFND 4-Step 위저드

1. **Step 1:** 마스터 ROI 드래그 지정
2. **Step 2:** 자릿수 입력 → 자동 등분 분할선 표시
3. **Step 3:** 가상 블록 오버레이 (두께/기울기/간격 슬라이더) 실시간 미리보기
4. **Step 4:** 판정 모드 선택 (○ Equal / ○ Range) + 목표값 입력

## 7. 레시피 저장/로드 시스템

### 7.1. 파일 포맷 `.hyv`

```
[HEADER 16B]  Magic(0xHY01, 2B) + Version(2B) + CRC32(4B) + ToolCnt(2B) + LogicCnt(2B) + Reserved(4B)
[CAMERA  32B] SET_CAMERA command
[TOOLS  N×32B] DFS 순서 SET_TOOL commands (비전툴)
[LOGIC  M×32B] ADD_LOGIC_NODE commands (로직툴)
[REFS   K×8B]  참조 엣지 (logic_node_id(2B) + ref_tool_id(2B) + pad(4B))
```

### 7.2. 파일명 및 저장 위치

- 슬롯 기반: `recipe_s1.hyv` ~ `recipe_s8.hyv`
- 물리 장치: Flash 내부 (SAVE_RECIPE 0x18 명령)
- 가상 장치(VVM): `./recipes/` 로컬 폴더

### 7.3. 동기화 흐름 (PC 주도)

```
[연결 시작]
PC → PING(0xF0) → 장치 응답
PC → recipe_s1.hyv 로드 → SET_TOOL 배치 전송 → 장치 트리 재조립

[저장]
[저장] 버튼 → PC: recipe_s1.hyv 갱신 + SAVE_RECIPE(0x18, slot=1) 전송

[장치 독립 실행, 미래]
전원 ON → Flash 슬롯 로드 → 자체 RUN
USB 연결 → 현재 실행 트리를 SET_TOOL Burst로 PC 브로드캐스트
```

### 7.4. UI 상태바 표시

- 💾 "미저장 변경사항 있음"
- ✅ "저장됨 / 장치 동기화됨"
- ⚠ "장치와 미동기화"

## 8. 엣지 케이스 방어 UX

| 케이스 | 방어 |
|--------|------|
| 로직툴에 자식 0개 | ⚠ 아이콘 표시. 빈 AND=True, 빈 OR=False |
| 비전툴을 트리 루트에 드롭 시도 | 드롭 거부 + 툴팁 "로직툴 내부에 배치해야 합니다" |
| 참조된 비전툴 삭제 (TEACH에서) | "이 툴은 N개 로직 참조에서 사용 중입니다. 삭제하면 해당 참조가 🚫로 표시됩니다. 계속?" |
| HyWhen watch_tool_id 자기 자신 | 드롭다운에서 자기 자신 제외 |
| 실행 순서 의존성 위반 | ⚠ 아이콘 + 툴팁 (실행은 허용) |
| 레시피 미저장 상태에서 모드 전환 | "저장되지 않은 변경사항이 있습니다" 다이얼로그 |
| 장치 연결 끊김 + TEACH/TEST | 자동 VVM Hot-Swap → 상태바 "VVM 모드" 표시 |

## 9. 사용 시나리오 시뮬레이션

### SC-01: 첫 HyLine 생성 (TEACH 설정)
1. TEACH 설정 서브모드 → 빈 리스트
2. `[+비전툴▼]` → HyLine → 캔버스 드래그 → HyLine #1 생성
3. 리스트 추가 + OverlayPanel 출현 + UI Preview Engine 결과 렌더링

### SC-02: ROI 회전 조정
1. HyLine #1 ROI 선택 → 상단 회전 핸들 드래그
2. 실시간 각도 수치 표시 + UI Preview Engine 즉시 재연산
3. 검증 서브모드 전환 → 동영상에서 검증 → 회전값 미세 조정
4. 설정 서브모드 복귀 → 조정된 회전값 그대로 반영

### SC-03: HyLinePatMat 타겟 라인 할당
1. HyLine #1 이미 존재
2. HyLinePatMat #2 생성 → OverlayPanel에서 `target_line` 드롭다운 → #1 HyLine 선택
3. 하이라이트: 리스트에서 #1이 파란 테두리 표시

### SC-04: TEST 모드 로직 조립
1. TEST 설정 서브모드 진입
2. `[+로직툴▼]` → HyAnd → 트리 루트에 HyAnd #10 추가
3. 비전툴 리스트에서 #1 HyLine 드래그 → HyAnd #10 노드에 드롭 → 🔗 #1 참조 노드 생성
4. 같은 방법으로 #2 HyLine 참조 추가
5. HyAnd #10 노드 클릭 → OverlayPanel에서 `[Fin]` 배지 토글 ON
6. `[HyFin 출력 설정]` 탭 → I/O 핀 매핑 설정

### SC-05: HyWhen 상태머신 (0.4 §3.3 ver→aaa→bbb)
